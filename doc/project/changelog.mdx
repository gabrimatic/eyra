Eyra's source changelog lives in `CHANGELOG.md`.

Current package metadata reports version `4.3.1`. The current release line is `4.3.1`.

The 4.3.0 line adds compact local memory through `mcp-prose-memory`, user-editable `AGENTS.md` and `personality.md`, memory controls in terminal, CLI, Web, and menu bar surfaces, plus setup and docs for the new local memory path.

The 4.3.1 line tightens memory write-confirmation behavior and upgrades memory readiness checks to probe real MCP compact-context access.

`## Unreleased` in `CHANGELOG.md` should stay empty or say there are no unreleased changes immediately after publishing 4.3.1, until new changes land.

## Update rules

When runtime behavior changes, keep these surfaces aligned:

- `CHANGELOG.md`
- `README.md`
- `.env.example`
- Mintlify docs
- Tests
- Certification notes when voice-to-computer behavior changes

## Release evidence

A release note should state which checks actually ran:

- Unit tests.
- Ruff.
- Lockfile check.
- Setup syntax.
- Package build.
- Runtime smoke.
- Certification matrix.
- Physical microphone challenge when claimed.
- Secret scan when public release scope requires it.
