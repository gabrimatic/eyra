---
title: "Installation"
description: "Install Eyra, configure the local runtime, and start the terminal session."
---

Runtime: macOS, Python 3.11+, Homebrew, `uv`, and an OpenAI-compatible provider. Preferred setup: Ollama at `localhost:11434` and Local Whisper for speech.

## Source checkout

```bash
git clone https://github.com/gabrimatic/eyra.git
cd eyra
chmod +x setup.sh && ./setup.sh
```

`setup.sh` creates `.env`, installs dependencies, checks the backend and models, checks Local Whisper when voice is enabled, and registers the `eyra` command. It preserves an existing `.env`, removes older shell lines ending in `# eyra`, then adds one current `~/.local/bin` PATH line.

Run from source:

```bash
uv run python src/main.py
```

After your shell sees `~/.local/bin`, run:

```bash
eyra
```

## Installer script

`install.sh` installs a release archive into a managed directory and runs non-interactive setup. For `v4.2.0`, use authenticated GitHub access while the repository is private.

```bash
curl -fsSL https://raw.githubusercontent.com/gabrimatic/eyra/master/install.sh | bash
```

Safety checks:

- Requires macOS on Apple Silicon.
- Requires Python 3.11+; installs `python@3.11` through Homebrew when possible.
- Installs `uv` when missing.
- Allows install directories only under `~/.local/share/eyra/`, `~/Applications/`, or `/opt/`.
- Refuses `/`, `$HOME`, and `~/.local` as install targets.
- Preserves existing installs by moving them to a timestamped backup.
- Rolls back the install directory if first-run setup fails.

Supported overrides:

```bash
EYRA_VERSION=v4.2.0 EYRA_INSTALL_DIR="$HOME/.local/share/eyra/app" ./install.sh
```

Private repositories require `GITHUB_TOKEN`. The installer reads GitHub release JSON, prefers release assets with checksum files, and refuses unverified tag-archive fallback unless you explicitly set `EYRA_ALLOW_UNVERIFIED_TAG_ARCHIVE=true`. For testing from a checked-out tree, use `EYRA_SOURCE_PATH=/path/to/eyra ./install.sh`; the installed app still writes user config to `~/.config/eyra/.env`, not into the app directory.

`eyra doctor --json` reports installer-managed app copies as `managed-install`.

## Homebrew tap

Current formula status: custom-tap scaffold.

```bash
brew install gabrimatic/eyra/eyra
```

The checked-in formula installs from the `master` branch and reports `4.2.0`. The formula remains tap preparation until it is switched to a tagged release URL and pinned sha256. Treat it as tap preparation, not a stable public formula.

Formula behavior:

- Runs `uv sync --frozen --no-dev` at install time.
- Command shims use `uv run --frozen --no-sync`.
- Provides `eyra`, `eyra-web`, `eyra-doctor`, `eyra-certify`, `eyra-setup`, and `eyra-connectors`.
- Runs `eyra version`, `eyra paths --json`, and `eyra doctor --json` in the formula test. The formula test expects a real configured backend.

## Local provider

Default provider: Ollama.

```env
API_BASE_URL=http://localhost:11434/v1
API_KEY=ollama
MODEL=gemma4:e4b
```

Eyra checks `/v1/models`, falls back to Ollama `/api/tags`, verifies every configured model, and auto-pulls missing Ollama models when `AUTO_PULL_MODELS=true`.

## Voice dependency

Eyra uses Local Whisper for both transcription and speech output.

```bash
brew tap gabrimatic/local-whisper
brew install local-whisper
```

Preflight resolves the `wh` binary from `PATH`, LaunchAgent plists, and common Homebrew paths. If voice is disabled, the Local Whisper check is skipped.

## Mock mode

Use mock mode when you want to smoke-test Eyra without a running model backend.

```bash
USE_MOCK_CLIENT=true LIVE_LISTENING_ENABLED=false LIVE_SPEECH_ENABLED=false uv run python src/main.py
```

Mock mode intentionally bypasses backend and model preflight. It still exercises the terminal session, command handling, local routing, and basic streaming path.

## Web UI

Standalone Web runtime:

```bash
WEB_UI_ENABLED=true uv run python -m web.server
```

Shared terminal and Web runtime:

```bash
WEB_UI_ENABLED=true eyra
```

Default host: `127.0.0.1`. Set `WEB_UI_HOST=0.0.0.0` only when you intentionally want another device on the network to connect.

## Verification

Run the standard local checks:

```bash
uv run pytest -q
uv run ruff check src tests
uv lock --check
bash -n setup.sh
```

Run certification when validating voice-to-computer behavior:

```bash
uv run python scripts/certify_voice_to_computer.py
```
