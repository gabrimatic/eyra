Eyra's source changelog lives in `CHANGELOG.md`.

Current package metadata reports version `4.2.1`. Current `main` contains the 4.2.0 routing, voice, installer, and connector work plus the 4.2.1 certification fix.

Current unreleased work improves guided setup for normal users: respectful plain-language setup text, optional Ollama and Local Whisper install help, safer service startup, recommended-model download when no local model exists, hidden cloud API-key entry, installer repair guidance, and `.env` preservation for the newer connector, task-store, voice, OCR, and hands-free settings.

Current `main` also contains 4.2.0 packaging work: a custom-tap Homebrew formula scaffold, `install.sh`, CLI support commands, source-checkout-only `.env` precedence, shell rc cleanup in setup/uninstall, installer safety checks, and matching certification rows.

Current `main` also contains universal connector workers: `CONNECTORS_*` settings, manifest validation, static transports for CLI/MCP/local HTTP/remote HTTP/browser-agent/coding-agent/guarded Python-module workers, acceptance states, connector terminal commands, connector CLI validation, authenticated Web connector APIs, approval-gated file mutation, cancellation, output caps, secret redaction, sandboxed working directories, job logs/artifacts, route trace metadata, and connector certification rows.

The 4.2.1 patch fixes synthetic microphone certification: `--synthetic-mic` now starts and stops `fake-mic` for virtual microphone checks instead of only labeling the result.

## Update rules

When runtime behavior changes, keep these surfaces aligned:

- `CHANGELOG.md`
- `README.md`
- `.env.example`
- Mintlify docs
- Tests
- Certification notes when voice-to-computer behavior changes

## Release evidence

A release note should state which checks actually ran:

- Unit tests.
- Ruff.
- Lockfile check.
- Setup syntax.
- Package build.
- Runtime smoke.
- Certification matrix.
- Physical microphone challenge when claimed.
- Secret scan when public release scope requires it.
